How does JS work ?

One of the most effective and widely used scripting languages is JavaScript, a client-side language. The phrase "client-side scripting language" refers to the fact that it executes client-side (or on the client machine) within web browsers. A Document Object Model (DOM) is made from the contents of a web page by the web browser after it has loaded it, parsed the HTML, and rendered the page. The DOM presents a live view of the web page to your JavaScript code. The browser will then download all files that are connected to the HTML, including CSS and image files. The CSS parser is where the CSS data is found. The DOM first assembles the HTML and CSS to build the web page. The JavaScript engine of the browsers then loads JavaScript files and inline code but does not instantly execute the code. It holds off until the HTML and CSS have fully loaded.The order here is important. If the JavaScript did not wait for the HTML and CSS to finish, it would not be able to change the DOM elements.

Local storage and Session Storage Difference ?

Across browser sessions, local storage preserves the storage. Thus, the storage is kept even if the browser is closed and then opened again or the website is reopened in a new tab. There is no time limit.
When the page session is over, the session storage is deleted. As long as the tab or the browser is open, a page session is active. As a result, whenever you open a new tab or window, a new session and new storage are created.
